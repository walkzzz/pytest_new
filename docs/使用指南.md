## 环境准备

### 系统要求
- **操作系统**: Windows 10/11
- **Python**: 3.10 或更高版本
- **内存**: 至少 4GB RAM
- **磁盘空间**: 至少 500MB 可用空间

### 安装步骤

1. **克隆项目**
   ```bash
   git clone https://github.com/walkzzz/pytest_new.git
   cd pytest_new
   ```

2. **创建虚拟环境（推荐）**
   ```bash
   python -m venv venv
   venv\Scripts\activate  # Windows
   ```

3. **安装依赖**
   ```bash
   pip install -r requirements.txt
   ```

4. **安装可选工具**
   ```bash
   # Allure 命令行工具（用于生成报告）
   # 下载地址: https://github.com/allure-framework/allure2/releases
   # 将 allure/bin 添加到 PATH 环境变量
   ```

5. **验证安装**
   ```bash
   python -m pytest --version
   python scripts/cli.py --help
   ```

## 项目配置

### 配置文件说明

项目使用多层配置系统：

1. **环境变量** (最高优先级)
2. **`.env` 文件** (项目根目录)
3. **YAML 配置文件** (`tests/configs/`)
4. **代码默认值** (最低优先级)

### 初始配置

1. **复制环境变量模板**
   ```bash
   copy .env.example .env
   ```

2. **编辑 `.env` 文件**
   ```env
   # 应用配置
   APP_NAME=YourApplication.exe
   APP_PATH=C:\Program Files\YourApp
   BACKEND=uia  # 或 win32
   
   # 日志配置
   LOG_LEVEL=INFO
   LOG_DIR=logs
   
   # 测试配置
   DEFAULT_TIMEOUT=30
   RETRY_COUNT=3
   ```

3. **创建测试配置**
   在 `tests/configs/` 目录下创建 YAML 配置文件，例如 `login.yaml`:
   ```yaml
   app:
     process_name: "${APP_NAME}"
     backend: "${BACKEND}"
     main_window:
       title: "登录窗口"
       state: "visible"
   
   controls:
     username_field:
       type: "textbox"
       locator:
         automation_id: "txtUsername"
         class_name: "Edit"
   
     password_field:
       type: "textbox"
       locator:
         automation_id: "txtPassword"
         class_name: "Edit"
   
   test_data:
     valid_user:
       username: "testuser@example.com"
       password: "Password123!"
   
   test_cases:
     login_success:
       description: "成功登录测试"
       steps:
         - name: "输入用户名"
           action:
             control: "username_field"
             method: "set_text"
             args: ["${test_data.valid_user.username}"]
   ```

### 基本测试用例

```python
# tests/test_login.py
import pytest
from src.core.base_test import BaseTest

class TestLogin(BaseTest):
    """登录功能测试"""
    
    @pytest.mark.smoke
    @pytest.mark.ui
    def test_valid_login(self):
        """有效用户名密码登录"""
        # 启动应用
        self.launch_app()
        
        # 执行登录
        self.login_page.login(
            username="testuser@example.com",
            password="Password123!"
        )
        
        # 验证登录成功
        assert self.home_page.is_visible()
        assert self.home_page.get_welcome_text() == "欢迎, testuser"
        
        # 截图
        self.take_screenshot("login_success")
    
    @pytest.mark.parametrize("username,password,expected_error", [
        ("", "password", "用户名不能为空"),
        ("user", "", "密码不能为空"),
        ("invalid", "wrong", "用户名或密码错误"),
    ])
    def test_invalid_login(self, username, password, expected_error):
        """无效登录测试"""
        self.launch_app()
        self.login_page.login(username, password)
        
        assert self.login_page.get_error_message() == expected_error
```

### 使用页面对象模式

```python
# tests/pages/login_page.py
from src.core.base_page import BasePage
from src.pywinauto.controls import TextBox, Button, Label

class LoginPage(BasePage):
    """登录页面对象"""
    
    def __init__(self, app):
        super().__init__(app)
        self.username = TextBox(
            app, 
            locator={"automation_id": "txtUsername"}
        )
        self.password = TextBox(
            app,
            locator={"automation_id": "txtPassword"}
        )
        self.login_button = Button(
            app,
            locator={"title": "登录", "class_name": "Button"}
        )
        self.error_label = Label(
            app,
            locator={"automation_id": "lblError"}
        )
    
    def login(self, username, password):
        """执行登录操作"""
        self.username.set_text(username)
        self.password.set_text(password)
        self.login_button.click()
    
    def get_error_message(self):
        """获取错误消息"""
        if self.error_label.is_visible():
            return self.error_label.get_text()
        return ""
    
    def clear_fields(self):
        """清空输入框"""
        self.username.set_text("")
        self.password.set_text("")
```

```python
import pytest
import json

# 从JSON文件加载测试数据
with open("tests/test_data/users.json") as f:
```

### Fixture 缓存优化

```python
import pytest
from functools import lru_cache

# 会话级缓存（推荐用于昂贵资源）
@pytest.fixture(scope="session")
def expensive_resource():
    """昂贵资源fixture（整个测试会话只初始化一次）"""
    print("初始化昂贵资源...")
    resource = create_expensive_resource()
    yield resource
    print("清理昂贵资源...")
    resource.cleanup()

# 使用 lru_cache 缓存计算结果
@pytest.fixture
@lru_cache(maxsize=None)
def cached_config():
    """缓存配置计算"""
    return load_and_process_config()

# 参数化fixture缓存
@pytest.fixture(params=["v1", "v2", "v3"])
def app_config(request):
    """参数化配置fixture"""
    config = load_config(request.param)
    request.addfinalizer(lambda: config.cleanup())
    return config

def test_with_cached_fixtures(expensive_resource, cached_config, app_config):
    """使用缓存fixture的测试"""
    # expensive_resource 在整个测试会话中只初始化一次
    # cached_config 使用LRU缓存
    # app_config 根据参数重复使用
    assert expensive_resource.is_ready()
    assert cached_config.is_valid()
    assert app_config.version in ["v1", "v2", "v3"]
```

### 自定义报告

```python
# conftest.py 中自定义报告钩子
import pytest
import json
from datetime import datetime

@pytest.hookimpl(tryfirst=True, hookwrapper=True)
def pytest_runtest_makereport(item, call):
    """自定义测试报告"""
    outcome = yield
    report = outcome.get_result()
    
    if report.when == "call" and report.failed:
        # 收集失败信息
        failure_info = {
            "test": item.name,
            "file": item.location[0],
            "line": item.location[1],
            "error": str(report.longrepr),
            "timestamp": datetime.now().isoformat()
        }
        
        # 保存到文件
        with open("failures.json", "a") as f:
            json.dump(failure_info, f)
            f.write("\n")
```

## CI/CD 集成

### GitHub Actions 配置

```yaml
# .github/workflows/test.yml

jobs:
  test:
    runs-on: windows-latest
    
    strategy:
      matrix:
        python-version: ["3.10", "3.11", "3.12"]
    
    steps:
    - uses: actions/checkout@v3
    
    - name: Set up Python ${{ matrix.python-version }}
      uses: actions/setup-python@v4
      with:
        python-version: ${{ matrix.python-version }}
    
    - name: Install dependencies
      run: |
        python -m pip install --upgrade pip
        pip install -r requirements.txt
    
    - name: Run tests with parallel optimization
      run: |
        pytest -n auto --dist loadgroup \
          --cov=src --cov-report=xml \
          --junitxml=junit-report.xml \
          --html=reports/report.html \
          --self-contained-html
    
    - name: Upload test results
      if: always()
      uses: actions/upload-artifact@v3
      with:
        name: test-results-${{ matrix.python-version }}
        path: |
          reports/
          junit-report.xml
          coverage.xml
```

## 最佳实践

### 测试组织

1. **按功能模块组织测试**
   ```
   tests/
   ├── auth/           # 认证相关测试
   │   ├── test_login.py
   │   ├── test_registration.py
   │   └── test_password.py
   ├── user/           # 用户管理测试
   │   ├── test_profile.py
   │   └── test_preferences.py
   └── admin/          # 管理功能测试
       ├── test_dashboard.py
       └── test_settings.py
   ```

2. **使用有意义的测试名称**
   ```python
   # 好：清晰表达测试意图
   def test_user_cannot_login_with_invalid_password():
   
   # 不好：模糊不清
   def test_login_1():
   ```

3. **保持测试独立**
   ```python
   # 好：每个测试独立运行
   def test_create_user():
   def test_delete_user():
   
   # 不好：测试间有依赖
   def test_create_user():
       user_id = create_user()  # 创建用户
   
   def test_update_user():
       update_user(user_id)     # 依赖前一个测试的user_id
   ```

### 断言策略

1. **使用描述性断言消息**
   ```python
   # 好：清晰的错误信息
   assert response.status == "success", \
       f"响应状态应为'success'，实际为'{response.status}'"
   
   # 不好：无信息量的错误
   assert response.status == "success"
   ```

2. **分层验证**
   ```python
   def test_complex_response():
       result = complex_operation()
       
       # 第一层：基本验证
       assert result is not None
       assert isinstance(result, dict)
       
       # 第二层：结构验证
       assert "data" in result
       assert "status" in result
       
       # 第三层：业务验证
       if result["status"] == "success":
           assert "user" in result["data"]
           assert result["data"]["user"]["id"] > 0
   ```

3. **使用自定义断言**
   ```python
   from src.pywinauto.assertions import assert_visible_with_timeout
   
   def test_ui_element_appears():
       # 自定义断言，包含超时等待
       assert_visible_with_timeout(
           element=login_button,
           timeout=10,
           message="登录按钮应在10秒内出现"
       )
   ```

   ```python
   # 模块级：测试数据，配置文件

   ```

2. **及时清理资源**

## 故障排除

### 常见问题解决

1. **应用启动失败**
   ```
   问题: Application().start() 失败
   解决:
   1. 确认应用路径正确
   2. 检查应用是否已安装
   3. 以管理员身份运行测试
   4. 检查应用依赖项
   ```

2. **控件找不到**
   ```
   问题: 无法找到控件元素
   解决:
   1. 使用 Inspect.exe 验证控件属性
   2. 调整定位策略（title → automation_id）
   3. 增加等待时间
   4. 检查应用 backend 设置（uia/win32）
   ```

3. **测试不稳定**
   ```
   问题: 测试有时成功有时失败
   解决:
   1. 增加等待和重试机制
   2. 检查资源竞争问题
   3. 确保测试独立性
   4. 使用稳定的定位器
   ```

4. **报告生成失败**
   ```
   问题: Allure/HTML 报告生成失败
   解决:
   1. 确认输出目录有写入权限
   2. 检查依赖是否正确安装
   3. 查看 pytest 日志输出
   4. 尝试清理旧报告文件
   ```
